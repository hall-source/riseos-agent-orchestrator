# Review And Synthesis Artifacts

## Purpose

This note documents the canonical mock governance artifacts for the Weekly Marketing Command Brief loop.

The governance chain is:

```text
specialist evidence
-> marketing reviewer packet
-> Clone Banks HQ synthesis memo
-> human approval readiness
```

This remains mock-only. It does not connect live Google Ads, HubSpot, GA4, Search Console, Slack, Monday, Drive, OpenAI, ChatGPT agents, or real worker execution.

## Reviewer Artifact

The mock reviewer step creates two records in Agent Bus:

1. A canonical Agent Bus review packet through `POST /review-packets`, then attaches it through `POST /work-items/{work_item_id}/review`.
2. A rich governance evidence artifact through `POST /evidence-packets`, then attaches it through `POST /work-items/{work_item_id}/evidence`.

The canonical review packet is lifecycle-shaped:

```json
{
  "work_item_id": "...",
  "reviewer": "hall-marketing-reviewer",
  "review_status": "approved",
  "risk_level": "low",
  "findings": [
    "Mock evidence is structurally complete for workflow testing.",
    "No live platform data was accessed.",
    "No external action is approved by this mock review."
  ],
  "required_changes": []
}
```

The rich reviewer artifact is attached as an evidence packet with `artifact_type=risk_review` and `evidence_type=risk_review` in `test_results`:

```json
{
  "artifact_type": "risk_review",
  "produced_by": "hall-marketing-reviewer",
  "workflow_type": "weekly_marketing_command_brief",
  "summary": "Mock reviewer validated that all specialist artifacts are mock-only and safe for workflow testing.",
  "checked": [
    "specialist evidence exists",
    "all evidence is marked mock_only",
    "live_platform_access is false",
    "no production actions are requested",
    "human approval remains required before execution"
  ],
  "findings": [
    "Mock evidence is structurally complete for workflow testing.",
    "No live platform data was accessed.",
    "No external action is approved by this mock review."
  ],
  "risk_flags": [
    "mock_only_no_business_decisions",
    "requires_real_data_before_operational_use"
  ],
  "approval_recommendation": "ready_for_hq_synthesis_mock_only",
  "confidence": "mock_only",
  "human_approval_required": true
}
```

## Summary Readiness

`GET /api/v1/marketing/workflows/{workflow_id}/summary` calculates readiness from actual Agent Bus state:

- Specialist evidence is complete when all four specialist work items exist and each has attached evidence.
- Review is complete when the reviewer item has an attached `risk_review` artifact, with legacy fallback to `metadata.review_packet_ids`.
- Synthesis is complete when the HQ item has an attached `synthesis_memo` artifact, with legacy fallback to synthesis packet metadata.
- Human approval is ready when specialist evidence, review, and synthesis are complete, approval is required, and no human approval has been recorded yet.

When human approval is ready, `missing=[]` and `next_action` is:

```text
Hall can review the mock HQ synthesis memo. No production action is allowed from mock evidence.
```

## Mock-Only Safeguards

Every governance artifact includes explicit safeguards:

```json
{
  "mode": "mock_only",
  "confidence": "mock_only",
  "mock_only": true,
  "live_platform_access": false
}
```

The mock reviewer also checks that no production action is requested and that human approval remains required before execution.

These safeguards exist so the MVP can prove the governance chain without creating any path from mock evidence to real business decisions or production platform writes.

## Remaining Work Before Real Agent Execution

Before real agent execution is allowed, the system still needs:

- a Marketing Agent Worker Adapter contract
- durable human approval action and audit trail
- source-by-source live integration gates
- production-safe permission boundaries for each marketing platform
- clear rollback and disable controls for worker execution
- tests that prove mock evidence cannot trigger production action

Until those exist, this workflow is a governance and visibility proof only.
